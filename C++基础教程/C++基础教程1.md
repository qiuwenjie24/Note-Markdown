# 绪论

## 简介

C++ 是一种**高性能**的**编译型**编程语言，是C语言的增强版，支持面向对象和泛型编程。

源文件扩展名： `.cpp`、`.cp` 或 `.c`。

**核心特点：**

1. **速度快** - 直接编译为机器码，运行效率接近C语言
2. **控制力强** - 允许直接操作内存和硬件
3. **静态类型** - 变量类型必须预先声明
4. **手动内存管理** - 需要自己分配和释放内存
5. **直接操作硬件**

**与Python的主要区别：**

- Python：写代码 → 直接运行
- C++：写代码 → 编译机器语言 → CPU运行生成的可执行文件

**应用场景：**

- 游戏开发、操作系统、浏览器、数据库等性能要求高的领域

**工作原理**：

C++将编写的代码传入编译器，编译器将其转换为机器码，CPU读取机器码，并执行相应的操作。

C++是一种原生语言，而C++编译器为目标平台和目标架构生成（只适用于该目标平台的）机器码，让目标系平台的CPU执行指令。

## 环境配置

- **文本编辑器**：用于编写和保存源代码，推荐使用 Visual Studio Code。
- **C++ 编译器**：用于将 源代码 转化为计算机可识别的 机器语言，推荐使用 MinGW-w64。

### Visual Studio Code

1. **官网下载安装**：`https://code.visualstudio.com/`
2. **切换语言**：打开软件，点击右边工具栏中的 `extensions(ctrl+shift+X)`打开拓展商店，搜索插件`Chinese`并安装，可以切换为中文语言。
3. **安装 C/C++ 扩展包**：在拓展商店搜索`C/C++`并安装（选择）。
4. **设置字体大小**：在`设置(ctrl+,)`中选择合适的字体大小。
5. **选择调试器**：（假设已经安装了C++编译器）首次运行C++文件时，软件会要求你选择调试器，选择`C++ (GDB/LLDB)` --> `g++.exe` 即可。VS Code会生成两个配置文件：`launch.json` 和 `tasks.json`，并根据配置文件编译和运行程序。
6. C++ 程序的源文件扩展名：`.cpp`、`.cp` 或 `.c` 。

**注意**：如果VS Code没有生成两个配置文件`launch.json` 和 `tasks.json`，那么每次运行都需要手动再次选择调试器。可以在文件所在的位置手动新建一个名为`.vscode`的文件夹，并在这个文件夹中添加配置文件，以此解决这个问题。

```reStructuredText
C:\Users\QWJ\Desktop\
├── .vscode/            ← （新建文件夹）
│   ├── launch.json     ← （配置文件）
│   └── tasks.json      ← （配置文件）
└── test.cpp            ← （cpp源文件）
```

`launch.json` 文件的内容：

```json
{
    "version": "0.2.0",
    "configurations": [
        

        {
            "name": "C/C++: g++.exe 构建和调试活动文件",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}\\${fileBasenameNoExtension}.exe",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${fileDirname}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "miDebuggerPath": "D:\\Cpp\\mingw64\\bin\\gdb.exe",   // ← 注意：这里的路径要对应实际安装的路径
            "setupCommands": [
                {
                    "description": "为 gdb 启用整齐打印",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                },
                {
                    "description": "将反汇编风格设置为 Intel",
                    "text": "-gdb-set disassembly-flavor intel",
                    "ignoreFailures": true
                }
            ],
            "preLaunchTask": "C/C++: g++.exe 生成活动文件"
        }
    ]
}
```

`tasks.json`文件的内容：

```json
{
    "tasks": [
        {
            "type": "cppbuild",
            "label": "C/C++: g++.exe 生成活动文件",
            "command": "D:\\Cpp\\mingw64\\bin\\g++.exe",   // ← 注意：这里的路径要对应实际安装的路径
            "args": [
                "-fdiagnostics-color=always",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}\\${fileBasenameNoExtension}.exe"
            ],
            "options": {
                "cwd": "${fileDirname}"
            },
            "problemMatcher": [
                "$gcc"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "detail": "调试器生成的任务。"
        }
    ],
    "version": "2.0.0"
}
```



### 其他

**环境冲突**：如果你之前安装了Anaconda，则系统默认的Shell为Anaconda PowerShell，可能会出现运行c++时会显示conda错误等一大堆说明，但不影响程序的运行。本质是电脑上多个开发环境共存的正常现象。

**解决方法**：修改VS Code默认终端设置

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 `Preferences: Open User Settings` 并选择
3. 在设置中搜索 `terminal integrated default profile windows`
4. 将其设置为 `Command Prompt` 或 `PowerShell`

### MinGW-w64

1. **官网**：`https://sourceforge.net/projects/mingw-w64/files/`
2. **下载**：进入官网 --> `Downloads` --> `Pre-built Toolchains` --> `MinGW-W64-builds` --> 点击相应的`Installation: GitHub` --> 点击下载`x86_64-15.2.0-release-win32-seh-ucrt-rt_v13-rev0.7z`，其中`15.2.0`当前下载的版本。
3. **解压安装**：将下载的压缩包解压并放置到合适的目录下，如`D:\Cpp\mingw64` 。
4. **配置环境变量**：在Windows搜索框里输入`环境变量` --> 选择`编辑系统环境变量` --> 点击`环境变量(N)` --> 在下面的`系统变量(S)`区域中，找到并选中名为 `Path` 的变量 --> 点击下方的`编辑` --> 点击右侧的`新建` --> 输入你的MinGW的`bin`目录路径，如`D:\Cpp\mingw64\bin` --> 点击`确定`
5. **验证**：打开命令提示符（CMD），输入 `g++ --version` ，如果没有显示错误则说明配置成功。

## 知识结构

C++学习可以分成四个层次：

```reStructuredText
┌─────────────────────────────────────┐
│  第四层：现代C++特性（先不管）        │  ← 匿名函数、智能指针、移动语义
├─────────────────────────────────────┤
│  第三层：STL标准库                   │  ← vector、string、算法
├─────────────────────────────────────┤
│  第二层：核心编程能力                 │  ← 指针、引用、函数、面向对象
├─────────────────────────────────────┤
│  第一层：语法基础                    │  ← 变量、类型、循环、判断、数组
└─────────────────────────────────────┘
```

笔记也是按这个结构，从下往上一层一层介绍。
