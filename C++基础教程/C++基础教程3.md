# 核心编程能力

## 函数

函数就是“打包好的代码块”，输入参数进去，函数会返回相应的值。

每个 C++ 程序都至少有一个函数，即主函数 main() 。

### 基本结构

 函数的基本结构：

```c++
返回值类型 函数名(参数列表) {
    // 函数体：执行的代码
    return 返回值;
}
```

例子：

```c++
int add(int a, int b) {
    int result = a + b;
    return result;
}
```

**注意**：`return` 的作用是立即退出函数（不执行后面语句），并返回值。

### 函数类型

根据返回值和参数分类，可以讲函数分类四种类型。

- **类型1：有返回值 + 有参数（最常用）**

  ```c++
  // 两个整数相加，返回结果
  int add(int x, int y) {
      return x + y;
  }
  
  // 使用
  int sum = add(3, 5);  // sum = 8
  ```

- **类型2：有返回值 + 无参数**

  ```c++
  // 返回π的值
  double getPi() {
      return 3.14159;
  }
  
  // 使用
  double pi = getPi();  // pi = 3.14159
  ```

- **类型3：无返回值 + 有参数（用 void）**

  ```c++
  // 打印问候语，不需要返回什么
  void greet(string name) {
      cout << "Hello, " << name << "!" << endl;
      // 没有 return，或者写 return;（不跟值）
  }
  
  // 使用
  greet("张三");  // 输出：Hello, 张三!
  ```

- **类型4：无返回值 + 无参数**

  ```c++
  // 打印分隔线
  void printLine() {
      cout << "--------------------" << endl;
  }
  
  // 使用
  printLine();  // 输出：--------------------
  ```

**注意**：无返回值时，必须使用 `void` 类型。若使用其他类型，如 `int` ，但又没有使用 `return` 返回值，那么编译器会返回一个垃圾值。

### 参数传递

调用函数时会创建一个副本，同时将传入参数的值传递给副本。不同的参数传递方式会对参数又不同的影响。

C++ 有三种参数传递方式：

- **传值（默认方式）**：将传入变量的值赋值给副本，变量与副本之间独立、互不影响（相当于复制，会占用内存）。
- **引用传递（用 `&`）**：让变量与副本共享同一个地址，此时副本改变也会影响原变量。
- **指针传递**：传入的变量是指针变量，指针变量的值是地址，因此副本也是一个指针变量，存储同一个地址；当地址对应的值发生改变时，存储这个地址的指针变量所对应的值也发生改变。（与引用传递相同效果）

注：对象也可以作为参数传进去。

例子：

```c++
#include <iostream>
using namespace std;

// 1. 传值：不修改原值
void byValue(int x) {
    x = 100;
}

// 2. 引用传递：修改原值
void byReference(int &x) {
    x = 100;
}

// 3. 指针传递：修改原值
void byPointer(int *x) {  // 参数是指针
    *x = 100;            // 通过指针修改原变量
}

int main() {
    int a = 10, b = 10, c = 10;
    
    byValue(a);
    byReference(b);
    byPointer(&c);
    
    cout << "传值: " << a << endl;      // 10（没变）
    cout << "引用: " << b << endl;      // 100（变了）
    cout << "指针: " << c << endl;      // 100（变了）
    
    return 0;
}
```

**传值的图解**：

```reStructuredText
调用前： a = 10
调用时： 创建副本 x = 10
函数内： x = 100（修改副本）
函数结束： 销毁 x
回到main： a 还是 10（原值不变）
```

指针和引用的知识在下一节才会详细介绍。

这里只需要知道如果不想修改原值，则使用传值的方式；如果想要修改原值，则使用引用传递的方式。

引用传递和指针传递之间存在区别，新手优先使用引用传递。

### 函数的声明和定义

**声明函数**是告诉编译器“有这个函数”。

**定义函数**是告诉编译器“函数具体做什么”。

**使用函数之前，需要先声明函数。**至于函数的定义，我们可以分为函数与主程序在同一个文件，和函数与主程序在不同文件的情况。

#### 同一个文件的情况

**使用函数之前，要么已经定义了函数，要么声明了函数**。

如果已经在前面定义了函数，可以不必声明。

如果在前面声明了函数，那么函数的定义可以放后头。

例子：

```c++
#include <iostream>
using namespace std;

// 声明：告诉编译器，后面会有个叫 add 的函数
int add(int a, int b);

int main() {
    int result = add(3, 5);  // 可以调用，因为已经声明了
    cout << result << endl;
    return 0;
}

// 定义：真正实现这个函数
int add(int a, int b) {
    return a + b;
}
```

#### 不同文件的情况

假设一个程序分成三个文件：

```reStructuredText
my_program/
├── main.cpp      // 主程序
├── math.cpp      // 函数的定义
└── math.h        // 函数的声明（头文件）
```

**文件1：`math.h`（头文件，存放声明）**

```c++
// math.h
#ifndef MATH_H
#define MATH_H

// 声明：告诉编译器这些函数存在
int add(int a, int b);
int subtract(int a, int b);
double multiply(double a, double b);

#endif
```

**文件2：`math.cpp`（源文件，存放定义）**

```c++
// math.cpp
#include "math.h"  // 引入声明

// 定义：真正实现函数
int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

double multiply(double a, double b) {
    return a * b;
}
```

**文件3：`main.cpp`（主程序）**

```c++
// main.cpp
#include <iostream>
#include "math.h"  // 引入声明，就能调用这些函数

using namespace std;

int main() {
    int sum = add(5, 3);        // 调用 math.cpp 里定义的函数
    int diff = subtract(5, 3);
    double product = multiply(4.5, 2.0);
    
    cout << "和: " << sum << endl;
    cout << "差: " << diff << endl;
    cout << "积: " << product << endl;
    
    return 0;
}
```

### 函数的默认参数

定义函数时，可以为参数列表中后边的每一个参数指定默认值。

调用函数时，如果未传递参数的值，则会使用默认值；如果指定了值，则会忽略默认值，使用传递的值。

```c++
void greet(string name, string prefix = "Hello") {
    cout << prefix << ", " << name << "!" << endl;
}

int main() {
    greet("张三");           // 输出：Hello, 张三!
    greet("李四", "Hi");    // 输出：Hi, 李四!
    return 0;
}
```

**注意**：默认参数必须从右向左连续

```c++
// 正确
void func(int a, int b = 10, int c = 20);

// 错误（不能跳过b）
void func(int a = 10, int b, int c = 20);  // ❌
```

### 函数重载

C++ 允许多个函数同名，只要**参数列表不同**。

**注意**：

- 只要参数个数不同，或参数类型不同，就可以函数重载。
- 返回值类型不同不能构成重载。

函数重载例子：

```c++
// 两个整数相加
int add(int a, int b) {
    return a + b;
}

// 三个整数相加
int add(int a, int b, int c) {
    return a + b + c;
}

// 两个浮点数相加
double add(double a, double b) {
    return a + b;
}

int main() {
    cout << add(3, 5) << endl;       // 调用第一个：8
    cout << add(3, 5, 7) << endl;    // 调用第二个：15
    cout << add(3.5, 2.5) << endl;   // 调用第三个：6.0
    return 0;
}
```

## 指针

**指针**是一个变量，其值为另一个变量的地址。指针 = 存储变量地址的变量。

- **普通变量**：直接存值
- **指针变量**：存地址，通过地址间接找到值

### 基本语法

#### 声明指针

在使用指针之前，需要进行声明。

指针变量声明的语法为：

```c++
int *p;      // p 是一个指向 int 类型的指针
double *dp;  // dp 是一个指向 double 类型的指针
char *cp;    // cp 是一个指向 char 类型的指针
```

- `*` 表示这是一个指针变量。
- `int*` 表示的是指向 int 的指针类型。
- `int* p` 和 `int *p` 和 `int * p` 都是等价的。
- 更推荐使用 `int* p` ，能够明显看出是定义了一个指针变量 `p` 。

#### 取地址符 `&` 

`&` 放在普通变量前面，获取这个普通变量的地址。

**仅用于普通变量，不用于指针变量。**

例子：

```c++
int a = 10;
int *p = &a;  // p 指向 a，即 p 存的是 a 的地址
```

#### 解引用符 `*` 

`*` 放在指针变量前面，获取指针指向的值。

**仅用于指针变量，不用于普通变量。**

例子：

```c++
int a = 10;
int *p = &a;

cout << p << endl;   // 输出 a 的地址（比如 0x7ffd5c3b4a3c，一个十六进制数）
cout << *p << endl;  // 输出 a 的值：10

*p = 20;             // 通过指针修改 a 的值
cout << a << endl;   // 输出 20
```

注意：`int *p = &a;` 应该理解成定义了一个指针变量 `p` ，然后把地址 `&a` 传给指针变量。而不是把地址赋值给指针所对应的值 `*p` 。

### 指针状态

 指针有三种状态：

| 状态     | 写法                  | 含义             | 能否使用       |
| :------- | :-------------------- | :--------------- | :------------- |
| 有效指针 | `int *p = &a;`        | 指向某个变量     | ✅ 可以         |
| 空指针   | `int *p = nullptr;`   | 指向"什么都没有" | ❌ 解引用会崩溃 |
| 野指针   | `int *p;`（未初始化） | 指向随机地址     | ❌ 危险！       |

`nullptr` 是 C++11 引入的空指针常量，表示"不指向任何地方"。

**铁律**：声明指针时，要么赋一个有效地址，要么赋 `nullptr`，永远不要留空。

### 指针运算与数组名

指针可以加减整数。

如：声明指针 `int* p` ，则 `p+1` 表示地址 `p` 加上 `sizeof(int)` 字节。与指针的类型有关。

对于一个数组，地址是连续的：

```reStructuredText
地址示例（假设 int 占4字节）：
arr[0] 地址: 1000
arr[1] 地址: 1004
arr[2] 地址: 1008

p = 1000
p + 1 = 1004（不是1001）
```

因此，可以通过指针加减的方式，依次获得数组元素值或地址：

```c++
int arr[5] = {10, 20, 30, 40, 50};
int *p = &arr[0];  // p 指向第一个元素

cout << *p << endl;     // 10
cout << *(p + 1) << endl; // 20
cout << *(p + 2) << endl; // 30

// 遍历数组
for (int i = 0; i < 5; i++) {
    cout << *(p + i) << " ";  // 10 20 30 40 50
}
```

注意：C++一个非常重要的特性：数组名就是指针，并且是指向第一个元素的指针。即 `arr[0]` 等价于 `*arr` ； `arr` 等价于 `&arr[0]` 。

**用指针遍历数组的例子**：

```c++
int arr[5] = {10, 20, 30, 40, 50};

// 方式1：用索引
for (int i = 0; i < 5; i++) {
    cout << arr[i] << " ";
}

// 方式2：用指针
for (int *p = arr; p < arr + 5; p++) {
    cout << *p << " ";
}
```

## 引用

（对比指针的区别）

**引用**就是给变量起一个别名，也就是说，它是某个已存在变量的另一个名字。

类比：你叫"张三"，朋友叫你"三哥"。不管叫哪个名字，喊的都是同一个人。

**注意**：

- 引用不是独立存在的，它必须依附于某个已存在的变量。
- 变量名相当于“本名”，引用相当于“外号”或“别名”。
- 一个变量可以被多个引用绑定，所有这些引用都是该变量的“别名”。
- 一旦把引用初始化为某个变量，就可以使用该引用名称或变量名称来指向变量。

### 基本语法

引用的语法如下：

```c++
int a = 10;
int &ref = a;   // ref 是 a 的引用（别名）
int& ref = a;   // 等价表述，两者无区别

ref = 20;       // 修改 ref，a 也变成 20
cout << a;      // 输出 20
```

- `&` 在这里表示“引用”，而不是“取地址”。
- `int &` 表示“指向 int 的引用类型”。
- `ref` 就是 `a` 的另一个名字，它们共用一个地址。

### 引用的三大规则

**规则1：定义时必须初始化，不可以为空 `nullptr` 。**

```c++
int a = 10;
int &ref = a;   // ✅ 正确：定义时初始化

int &ref2;      // ❌ 错误：引用必须初始化
```

**规则2：一旦绑定，不能改变**

```c++
int a = 10, b = 20;
int &ref = a;   // ref 是 a 的引用

ref = b;        // 这是把 b 的值赋给 a，不是让 ref 改为引用 b
                // 执行后 a = 20, b = 20，但 ref 仍然是 a 的引用

cout << &a << endl;   // 地址 X
cout << &ref << endl; // 地址 X（相同）
cout << &b << endl;   // 地址 Y（不同）
```

**规则3：引用必须绑定到有效变量**

```c++
int &ref = 10;   // ❌ 错误：不能绑定到常量字面量

int a = 10;
int &ref = a;    // ✅ 正确：绑定到变量
```

### 引用作为函数参数（最重要用法）

引用最常用的场景是**函数参数**，可以修改外面的变量，同时语法比指针简洁。

在函数那一节介绍过，这里再以例子展示函数中三种传参的区别：

```c++
#include <iostream>
using namespace std;

// 传值：不修改原变量
void changeByValue(int x) {
    x = 100;
}

// 引用传递：修改原变量（语法简洁）
void changeByReference(int &x) {
    x = 100;
}

// 指针传递：修改原变量（语法繁琐）
void changeByPointer(int *x) {
    *x = 100;
}

int main() {
    int a = 10, b = 10, c = 10;
    
    changeByValue(a);
    changeByReference(b);
    changeByPointer(&c);
    
    cout << "传值后 a: " << a << endl;       // 10（没变）
    cout << "引用后 b: " << b << endl;       // 100（变了）
    cout << "指针后 c: " << c << endl;       // 100（变了）
    
    return 0;
}
```

### const 引用（常引用）

普通引用不能绑定到字面量，但是 `const` 引用可以绑定到临时变量和字面量，常用于函数参数。

==不太理解这个的用法，故可略过这部分==

例子：

```c++
// 普通引用：不能绑定到字面量
int &r1 = 10;   // ❌ 错误

// const 引用：可以绑定到字面量
const int &r2 = 10;  // ✅ 正确

// 函数参数中使用 const 引用（避免复制，同时保证不修改）
void printValue(const int &x) {
    cout << x << endl;
    // x = 100;  // ❌ 错误：x 是 const，不能修改
}

int main() {
    printValue(10);      // 可以直接传字面量
    int a = 20;
    printValue(a);       // 也可以传变量
    return 0;
}
```

**const 引用的好处**：

- 不会复制原变量（高效）
- 保证不修改原变量（安全）
- 可以接受字面量和临时变量（灵活）

### 引用和指针对比

**引用 vs 指针**

| 特性         | 引用                                | 指针                             |
| :----------- | :---------------------------------- | :------------------------------- |
| 本质         | 别名                                | 存地址的变量                     |
| 语法         | `int &ref = a;`，像普通变量一样使用 | `int *p = &a;`， 需要 `*` 和 `&` |
| 是否可为空   | ❌ 必须初始化                        | ✅ 可以为 `nullptr`               |
| 能否重新绑定 | ❌ 不能                              | ✅ 可以                           |
| 访问值       | 直接用 `ref`                        | 用 `*p`                          |
| 取地址       | `&ref` 得到原变量地址               | `p` 就是地址                     |
| 安全性       | 高                                  | 低（容易出错）                   |

例子：

```c++
// 对比示例
int a = 10, b = 20;

// 引用
int &ref = a;    // ref 是 a 的别名
ref = b;         // 把 b 的值赋给 a（a 变成 20），ref 仍然是 a 的引用
// ref = b; 不能改为引用 b

// 指针
int *p = &a;     // p 指向 a
p = &b;          // p 改为指向 b（可以重新绑定）
*p = 100;        // 修改 b 为 100
```

### 栈和堆概念

程序运行时，内存分为几个区域，最重要的是**栈**和**堆**：

```reStructuredText
┌─────────────────────────────────────────────────────────────────┐
│                        内存布局                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                        栈                               │   │
│   │  ┌─────┐ ┌─────┐ ┌─────┐                               │   │
│   │  │ int │ │ int │ │ char│  ← 局部变量、函数参数           │   │
│   │  │ a=10│ │ b=20│ │ c='A'                              │   │
│   │  └─────┘ └─────┘ └─────┘                               │   │
│   │                                                         │   │
│   │  特点：自动管理，大小固定，速度快                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                        堆                               │   │
│   │  ┌─────────┐ ┌─────────┐                               │   │
│   │  │ new int │ │ new int │  ← 手动分配的内存               │   │
│   │  │   (10)  │ │   (20)  │                               │   │
│   │  └─────────┘ └─────────┘                               │   │
│   │                                                         │   │
│   │  特点：手动管理，大小灵活，速度较慢                       │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**栈 vs 堆**：

| 特性     | 栈（Stack）              | 堆（Heap）                           |
| :------- | :----------------------- | :----------------------------------- |
| 管理方式 | 自动（函数结束自动释放） | 手动（必须用 delete 释放）           |
| 大小     | 固定（通常 1-8MB）       | 大（受系统内存限制）                 |
| 速度     | 快                       | 较慢                                 |
| 生命周期 | 随函数结束而结束         | 直到手动释放                         |
| 适用场景 | 小变量、数组大小固定     | ==大数组、大小不确定、需要长期存在== |

### 栈上的变量（自动管理）

到目前为止，我们用的都是栈上的变量，结束后自动释放内存。

例如：

```c++
void func() {
    int a = 10;        // 栈上分配
    int arr[100];      // 栈上分配
    // 函数结束时，a 和 arr 自动释放
}
```

**缺点**：栈的大小有限。如果分配过大的数组，会栈溢出：

```c++
int main() {
    int hugeArray[10000000];  // 约 40MB，可能栈溢出！
    return 0;
}
```

**栈溢出的结果**：程序会立即崩溃，操作系统会终止程序运行，并通常给出一个错误提示（如"段错误"或"栈溢出"）。

### 堆上的变量（手动管理）

### 分配变量

**使用 `new` 可以在堆上分配内存**：

```c++
int *p = new int(10);
```

1. 在栈上创建 `int` 类型的指针变量 `p`；
2. 在堆上创建 `int` 对象；
3. 将这个对象的地址传给指针 `p`；
4. 将数值 `10` 赋值给堆变量。

**注意**：在堆上分配变量时，我们**必须用指针变量**来保存返回的地址。因为堆内存没有名字，只能借助指针，通过地址访问。但指针变量本身在栈上，它指向的内存块在堆上。

**使用 `delete` 可以释放堆上的内存**：

```c++
// 释放内存
delete p;
    
// 释放后 p 变成野指针，最好设为 nullptr
p = nullptr;
```

**例子**：

```c++
#include <iostream>
using namespace std;

int main() {
    // 在堆上分配一个 int
    int *p = new int;     // 分配内存，返回地址
    *p = 10;              // 赋值
    cout << *p << endl;   // 10
    
    // 释放内存
    delete p;
    
    // 释放后 p 变成野指针，最好设为 nullptr
    p = nullptr;
    
    return 0;
}
```

#### 初始化

**三种（等价）初始化方式**：

```c++
// 方式1：先分配，再赋值
int *p1 = new int;
*p1 = 10;

// 方式2：分配时初始化
int *p2 = new int(10);   // 直接赋值为 10

// 方式3：统一初始化（C++11）
int *p3 = new int{10};
```



#### 分配数组

在堆上分配数组和分配变量的方式有些不同。

分配数组的语法： `new 类型[大小]` 

释放数组内存的语法：`delete[] 数组;`

```c++
#include <iostream>
using namespace std;

int main() {
    // 在堆上分配数组
    int size = 1000;
    int *arr = new int[size];  // 大小可以是变量！
    
    // 使用数组
    for (int i = 0; i < size; i++) {
        arr[i] = i;
    }
    
    // 释放数组（注意 []
    delete[] arr;
    arr = nullptr;
    
    return 0;
}
```

### 内存泄漏（Memory Leak）

**内存泄漏**：是指分配了堆内存，但忘记释放。

在堆上分配内存时，储存该内存地址的指针 `p` 是在栈上的。

当函数结束时，指针 `p` 本身被销毁，但 p 指向的堆内存还在，再也无法访问。

每次调用这个函数都会泄漏内存（即被内存被占用但无法访问）。

程序长时间运行，内存越占越多，最终内存耗尽，程序崩溃。

**避免方法**：

1. 每个 `new` 都要对应一个 `delete`
2. 释放后将指针设为 `nullptr`
3. 使用智能指针（现代 C++，以后会学）

### 悬空指针（Dangling Pointer）

**悬空指针**：释放内存后，指针还指向原来的地址（但那块内存已经还给系统了）。

**好习惯**：释放内存后立即将指针置空。

例子：

```c++
int *p = new int(10);
delete p;      // 内存释放
// p 仍然指向原来的地址，但该地址已无效

// *p = 20;    // 危险！访问已释放的内存

p = nullptr;   // 好习惯：释放后立即置空
```



## 类与对象

（封装、构造函数、成员函数）面向对象的起点

面向对象就是把数据和操作数据的函数打包成一个整体（对象），通过对象之间的交互来解决问题。

现实世界是由无数"物体"组成的——汽车会跑，猫会叫，人会工作。面向对象就是把这种思维方式搬到编程中。

**类**是一种封装了数据和函数的数据结构，它允许定义成员变量和成员函数，本质是自定义了一种新类型。

简单地说，类就是把数据和操作数据的函数打包在一起，并控制谁能访问它们。

类比：类就像一份产品设计蓝图，定义了产品的属性（数据）和功能（函数），但蓝图本身不是产品，需要用蓝图造出产品（对象）才能使用。例如，房屋建筑设计图是类，根据蓝图建造出来的房子是对象。

**类就是"自定义类型"，对象就是"根据类创建的变量"。**

### 类的定义及初始化

#### 定义

定义类的语法框架如下：

```c++
// 定义一个新的类型，名叫 ClassName
class ClassName {
private:
    // 私有成员：只能被类内部访问
    int privateData;
    
protected:
    // 受保护成员：可被子类访问
    int protectedData;
    
public:
    // 公有成员：所有人都能访问
    int publicData;  // 成员变量
    
    // 成员函数（方法）
    void func() {
        cout << "Hello" << endl;
    }
};  // 注意：分号不能少！
```

**注意**：

- 定义类的后面必须有分号 `;` 。
- **类的定义本身就包含了声明，无需额外再声明**。也因此类的定义必须放在前头或头文件里。
- 除了类，结构体的定义也是需要在代码块 `{}` 后有分号。此外的其他代码块都不需要分号。
- 这是由于C语言因为一些原因需要分号，而C++ 继承自 C 语言，保持了这种语法风格。

#### 初始化

初始化的语法：

```c++
// 创建对象
类名 对象名(参数);

// 调用成员函数
对象名.函数名(参数);
```

### 成员访问权限

类通过 `private` 、`protected` 和 `public` 三种类型控制类中成员变量和方法的访问。

| 访问修饰符  | 类内部 | 子类 | 外部 |
| :---------- | :----- | :--- | :--- |
| `private`   | ✅      | ❌    | ❌    |
| `protected` | ✅      | ✅    | ❌    |
| `public`    | ✅      | ✅    | ✅    |

- `private`：保护数据，不让成员被外部访问，仅可内部访问。
- `public `：允许成员被外部访问和修改。
- `protected`与`private` 的区别是，当这个类作为父类被其他类（子类）继承时，子类能否访问。继承的知识在下一节讲解。
- 习惯上，把成员变量放在 `private` 中，成员函数放在 `public ` 中。

### 类成员及其访问

类不仅可以包含数据（成员变量），还可以包含操作这些数据的函数（成员函数）。

通过创建类，可以访问类中的数据和方法。

例如：

```c++
#include <iostream>
using namespace std;

// 定义类
class Student {
private:
    int age;  // 私有成员变量

public:   
    double score; // 公有变量
    
    // 用来修改 private 变量的值
    void setAge(int a) {
        age = a;
    }
    
    // 用来读取 private 变量的值
    int getAge() {
        return age;
    }

};

int main() {
    // 创建对象（实例化）
    Student s1;
    
    // 访问成员（用点号 .）
    s1.score = 95.5;  // public 变量可以直接修改
    cout << s1.score; // 直接读取变量
    
    // s1.age = 18;  // ❌ age 是 private，不可以外部访问和修改
    s1.setAge(18);       // ✅ 通过函数去修改私有变量
    cout << s1.getAge();  // ✅ 通过函数去读取私有变量
    
    return 0;
}
```

### 构造函数

类中除了普通的成员函数，还可以有构造函数。

**构造函数**：创建对象时自动调用（运行）的特殊函数。与普通成员函数不同，无法手动调用。

**特点**：

- 创建对象时自动调用。
- 无法手动调用。
- 函数名必须和类名相同。
- 没有返回值类型（连 `void` 都不写）。

**作用**：用于初始化对象。例如，在创建对象时，有时候希望让类先跑一段代码，得到一些初始化好的数据，以便后续调用成员函数时，可以直接使用这些数据。构造函数就是用于封装这段代码。

例子：

```c++
#include <iostream>
using namespace std;

class Student {
public:
    int age;      // 年龄
    double score; // 成绩
    
    // 构造函数（和类同名，没有返回值）
    // 创建对象的同时，自动给成员变量赋初始值。
    Student(int a, double s) {
        age = a;
        score = s;
        cout << "学生已创建" << endl;
    }
};

int main() {
    Student s1(18, 95.5);  // 创建对象，自动调用构造函数
    cout << s1.age << endl;   // 18
    cout << s1.score << endl; // 95.5
    
    return 0;
}
```

区别：

- `struct`：只有数据，没有方法
- `class`：有数据和行为，需要封装

## 继承（面向对象）

继承就是让 一个类（子类）可以拥有另一个类（父类）的属性和方法，并可以添加自己的新属性和方法，实现代码复用。

父类也称为基类，基于基类而产生的类（子类）也称为派生类。

类比：儿子继承了父亲的基因——儿子有父亲的特征，还可以有自己的新特征。

### 继承的语法

基本语法：

```c++
class 子类名 : public 父类名 {
    // 子类新增的成员
};
```

子类不仅拥有父类的成员，此外还可以：

- 添加新的成员变量
- 添加新的成员函数

例如：已经有了哺乳动物（父类）的特征，而小狗是哺乳动物，我们就可以在哺乳动物的基础上，写小狗（子类）的特征。

```c++
#include <iostream>
#include <string>
using namespace std;

// 父类（基类）：通用的特征
class Animal {
public:
    string name;

    void eat() { cout << "吃东西" << endl; }
};

// 子类（派生类）：继承 Animal，再加自己的特征
class Dog : public Animal {
public:
    // 新增成员变量
    string breed;   // 品种
    
    // 新增成员函数
    void bark() { cout << "汪汪" << endl; }
    void showBreed() { cout << "品种: " << breed << endl; }
};

int main() {
    Dog d;
    
    // 从父类继承
    d.name = "旺财";   
    d.eat();   // 旺财 正在吃东西 
    
    // 子类自己的成员
    d.breed = "金毛";  
    d.bark();  // 旺财 汪汪叫
    d.showBreed();  // 旺财 品种：金毛
    
    return 0;
}
```

### 访问控制

子类能访问父类的哪些成员，取决于父类中成员的访问修饰符：

| 父类中的修饰符 | 子类能否访问 |
| :------------- | :----------- |
| `public`       | ✅ 可以       |
| `protected`    | ✅ 可以       |
| `private`      | ❌ 不可以     |

**记忆口诀**：

- `private`：只有自己（父类自己）能用
- `protected`：自己和儿子（子类）能用
- `public`：谁都能用

### 覆盖（重写）

子类的函数可以覆盖（重写）父类的同名同类型的函数（返回值类型、参数列表都要相同）。

因此，如果子类不满意父类的某个函数，可以自己重新写。

例如：

```c++
#include <iostream>
using namespace std;

class Animal {
public:
    void speak() {
        cout << "动物发出声音" << endl;
    }
};

class Dog : public Animal {
public:
    // 覆盖父类的 speak 函数
    void speak() {
        cout << "汪汪" << endl;
    }
};

int main() {
    Dog d;
    d.speak();  // 输出：汪汪（调用的是 Dog 自己的版本）
    return 0;
}
```

### 构造函数和继承

创建子类对象时：

1. 先调用父类的构造函数
2. 再调用子类的构造函数

先调父类的原因：子类依赖于父类的部分，必须先初始化好父类，才能初始化子类。

## 多态

多态允许同一个接口（函数名、操作符等）在不同对象上表现出不同的行为，从而让代码更灵活、可扩展。

通俗地说：**“一个接口，多种实现”**

### 动机

没有多态，想要根据不同的对象类型执行不同的行为，通常需要借助条件判断

```c++
#include <iostream>
using namespace std;

class Animal {
public:
    void speak() { cout << "动物发出声音" << endl; }
};

class Dog : public Animal {
public:
    void speak() { cout << "汪汪" << endl; }
};

class Cat : public Animal {
public:
    void speak() { cout << "喵喵" << endl; }
};

// 没有多态：需要手工判断类型
void makeSound(Animal& animal, int type) {
    if (type == 1) {           // 1 代表 Dog
        Dog& d = (Dog&)animal; // 强制转换
        d.speak();
    } else if (type == 2) {    // 2 代表 Cat
        Cat& c = (Cat&)animal;
        c.speak();
    } else {
        animal.speak();
    }
    // 每增加一种动物，都要修改这个函数！
}

int main() {
    Dog d;
    Cat c;
    
    makeSound(d, 1);  // 汪汪（要手动告诉函数：这是 Dog）
    makeSound(c, 2);  // 喵喵（要手动告诉函数：这是 Cat）
    
    return 0;
}
```

多态让编译器在运行时根据对象的实际类型决定调用哪个函数（动态绑定）：

```c++
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() { cout << "动物发出声音" << endl; } // 虚函数
};

class Dog : public Animal {
public:
    void speak() override { cout << "汪汪" << endl; }
};

class Cat : public Animal {
public:
    void speak() override { cout << "喵喵" << endl; }
};

// 统一接口：接受基类引用
void makeSound(Animal& animal) {
    animal.speak();  // 动态绑定：具体调用哪个 speak() 由传入的实际对象类型决定
}

int main() {
    Dog d;
    Cat c;
    
    makeSound(d);  // 汪汪
    makeSound(c);  // 喵喵
}
```

### 多态的分类

C++ 支持两种主要的多态形式：

| 类型         | 实现方式                   | 绑定时机 | 别名               |
| :----------- | :------------------------- | :------- | :----------------- |
| **静态多态** | 函数重载、运算符重载、模板 | 编译时   | 编译时多态、早绑定 |
| **动态多态** | 继承 + 虚函数（`virtual`） | 运行时   | 运行时多态、晚绑定 |
